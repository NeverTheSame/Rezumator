# Rezumator: Agent Notes (read this first)

If you're an agent touching this repo, welcome. This project generates a **fictional exemplar** resume + cover letter from a resume JSON and a job posting. It is allergic to secrets, hardcoded prompts, and “it worked on my machine”.

## Repo layout (new-ish)

- `cmd/rezumator/`: CLI entrypoint (the `main()` you can run/build)
- `*.go` in repo root: library code in package `rezumator`
- `tests/`: ALL tests live here (yes, all of them)
- `my_resumes/`: contains only a public placeholder in git

Build/run:
- `go run ./cmd/rezumator`
- `go build ./cmd/rezumator`

## Prompt management (non-negotiable)

**CRITICAL RULE:** All LLM prompts MUST live in `prompts.json`. Do NOT hardcode prompts in `.go` files.

Why:
- Centralized prompt auditability
- Easy iteration without recompiling
- Less “mystery meat” strings in code reviews

Implementation rules:
1. **No hardcoded prompts in `.go`** (system/user prompt text must come from `prompts.json`)
2. **Load via `loadPrompts()`** (see `prompts.go`)
3. Each prompt must have:
   - `system_prompt`
   - `user_prompt`

✅ Correct:
```go
prompts, err := loadPrompts()
if err != nil { return err }
p := prompts["job_extractor"]
out, err := client.GenerateContent(ctx, p.SystemPrompt, p.UserPrompt, input)
```

❌ Incorrect:
```go
systemPrompt := "You are a job requirements extractor..."
out, err := client.GenerateContent(ctx, systemPrompt, userPrompt, input)
```

Current prompt keys:
- `rezumator`: resume tailoring
- `cover_letter`: cover letter generation
- `job_extractor`: extract structured job requirements from `job_details.txt`

## LLM-only content rule

**Nothing user-facing should be generated by code templates.** If the LLM fails or output is weak, the run should fail (and logs should explain why). No “helpful” deterministic fallbacks that secretly write the resume for the model.

The only acceptable code-side transformations are:
- parsing
- normalization/format enforcement
- retries / revision passes (LLM writes the content)

## Tests rule: everything in `/tests`

Go won’t let `/tests` import a `package main` program, which is why we refactored to `package rezumator` + `cmd/rezumator`.

`test_exports.go` exists to expose a few internal helpers to `/tests` (because unexported functions don’t cross package boundaries). It is not a runtime feature; it’s scaffolding so tests can live where you want them.

## Public git safety (please don’t leak your own credentials)

Never commit:
- `secret` / `.env` / tokens
- `secret copy` (if it exists, it contains real keys — delete or ignore it)
- logs / `.azure/`
- real resumes (only placeholders)

Allowed in git:
- `my_resumes/sre_placeholder.json` (public-safe schema placeholder)

## Job input quality

`job_details.txt` is a template file. Better copy-paste = better matching. The job extractor prefers the full posting with responsibilities + requirements + tool names.

## Tone

Touch funny is fine. Breaking formatting rules is not. If you must choose, pick correctness over comedy.
